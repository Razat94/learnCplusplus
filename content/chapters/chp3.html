Variables in C++

Now that you've had a taste of what outputting things to the console is like, we can now begin learning programming. We do so by introducing the concept of variables

Variables are portions of memory that are used to store a value. Each variable needs to be declared with a unique name and data type. Variables are common through out all programming languages, and it would be pretty tough to write any programs without them. As previously stated in the last chapter, a variable definition tells the compiler where and how much storage to create for the variable. For example, a string variable will obviously hold more space than a char variable would. To initialize the variable, you would use the following syntax.

variable_name variable_type

Let's give an example to explain what I'm talking about.

Let's go over the program. We can visualize variables as boxes of memory.

First, we create or "declare" the variable:

blah

Then, assign a value to that variable using the assignment operator '=':

blah
← 5

The variable is now updated

blah
5

As you may know, variables are simply containers of data. But given the name variable, we can have that value change itself as well.

Here, we first create or "declare" the variable:

blah

Then, assign a value to that variable:

blah
← 5

Output the value of the variable

blah
5

BUT now we assign the value '10' to that variable:

blah
5 ← 10

Output the value of the variable

blah
10

We can also have a variable assign it's own value to itself.

In the program above, it first assigns itself the value of 5. It then assigns the value of blah + 10 to blah. Meaning, it calculates the value of blah + 5 and shoves that value into blah. It lastly assigns the value of blah + 5 to blah.

Things to look out for:
  • The name of a variable can be composed of letters, digits, and the underscore character. It must begin with either a letter or an underscore.
  • C++ is case-sensitive; consider the variables blah and BLAH. C++ will consider these variables as two seperate entities.
  • Don't name a variable by a reserved C++ keyword. You can't declare a variable int string.
  • Also, try not to redeclare a variable. So if at one point in time you have a variable 'x', don't use the statement 'int x = 5' at a later point in time.

In short, we can name variables however we like, but they must not clash with C++ rules.
